# Prompt vs Skill ใน GitHub Copilot (Workshop Note)

เอกสารนี้ใช้สำหรับอธิบายใน workshop ว่า Prompt และ Skill ต่างกันอย่างไร และสอนเดโมสั้นในห้องเรียน

## สรุปสั้น

- Prompt คือคำสั่งสำเร็จรูปที่เรียกใช้งานตรง ๆ
- Skill คือความสามารถแบบ workflow ที่ Agent ใช้ทำงานหลายขั้นตอนตามกติกา
- ในบางกรณี Skill อาจแสดงในรายการ slash command ได้ จึงดูเหมือน Prompt

## ตารางเทียบ Prompt vs Skill

| ประเด็น | Prompt | Skill |
|---|---|---|
| หน้าที่หลัก | คำสั่งสำเร็จรูปสำหรับงานเฉพาะ | ความสามารถเชิง workflow สำหรับงานหลายขั้น |
| การเรียกใช้ | มักเรียกผ่าน slash ได้ชัดเจน | เรียกอัตโนมัติตามบริบท และบางกรณีเรียกผ่าน slash ได้ |
| โครงสร้างไฟล์ | เน้นแม่แบบคำสั่ง | เน้นขั้นตอน วิธีคิด กติกา |
| ความเหมาะสม | งานสั้น ชัดเจน อินพุตตรง | งานซับซ้อน มีการอ่านหลายไฟล์และตัดสินใจหลายจุด |
| ตัวอย่างใน workshop นี้ | create-tc, review-coverage | create-scenario |
| ผลลัพธ์ที่คาดหวัง | ได้ output ตาม template | ได้ผลลัพธ์พร้อม reasoning ตามกติกาของ skill |
| ข้อควรรู้เวลาอธิบาย | ผู้เรียนจะเห็นเป็นคำสั่งเรียกตรง | ผู้เรียนควรเข้าใจว่าเป็นความสามารถ ไม่ใช่แค่ปุ่มคำสั่ง |

## ประโยคสรุปสำหรับพูดหน้า class

Skill ไม่ได้เป็นแค่ Prompt ที่ยาวขึ้น แต่เป็นชุดความสามารถที่ Agent ใช้ตัดสินใจทำงานหลายขั้นให้ครบตามกติกา
